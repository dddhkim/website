# 🚀 1인 기업 웹사이트 구축 핵심 요약 노트

지금까지 단 몇 시간 만에 두 개의 웹사이트를 밑바닥부터 만들고, 실제 인터넷에 배포하는 자동화 환경까지 구축하셨습니다. 대표님께서 거치신 **'실제 IT 스타트업 대표들이 경험하는 핵심 워크플로우'**를 이해하기 쉽게 요약해 드립니다.

---

## 1. 우리가 만든 웹사이트의 3대 요소 (Frontend)
웹사이트는 사실 복잡해 보이지만, **딱 3가지 파일**의 조화로 이루어집니다.

| 파일 종류 | 역할 (비유) | 우리가 한 작업 |
| :--- | :--- | :--- |
| **`index.html`** | **건물의 뼈대** | 글씨를 쓰고, 이미지가 들어갈 공간(빈 박스)을 만들고, 영역을 나누었습니다. |
| **`styles.css`** | **인테리어 디자인** | 다크모드, 크림색 배경, 그림자 효과, 글꼴 등 예쁘게 색칠하고 꾸몄습니다. |
| **`script.js`** | **전기와 수도 (움직임)** | 스크롤을 내릴 때 글씨가 스르륵 나타나는(Fade-in) 애니메이션(생명)을 불어넣었습니다. |

> [!NOTE]
> 이렇게 3가지 파일로 이루어진 사이트를 구경만 가능한 **'정적 웹사이트(Static Website)'**라고 부릅니다. 명함, 포트폴리오, 브랜드 소개 페이지에 가장 완벽한 형태입니다.

### 📦 GitHub (창고)
*   **역할**: 코드(파일)를 인터넷상에 안전하게 백업 및 보관하는 클라우드 창고

### 🏭 Vercel (클라우드 배포 공장)
*   **역할**: 깃허브 창고를 24시간 감시하고 있다가, 창고에 내용이 바뀌면 **알아서 웹사이트로 찍어내 주는 공장**
*   **장점**: 한 번 연결(Import)만 해두면 절대 다시 설정할 필요가 없습니다. 대표님이 저(AI)에게 코드를 고쳐달라고 해서 제가 깃허브에 밀어 넣기만 하면, Vercel이 알아서 10초 만에 인터넷 사이트를 갈아 끼웁니다!

## 3. 대표님이 앞으로 알아두시면 좋은 팁 💡

> [!TIP]
> **하나의 큰 창고에서 여러 사이트 뽑아내기**
> Vercel에서 하나의 `website` 깃허브 창고를 선택한 뒤, **`Root Directory`** 만 `portfolio` 혹은 `lumiere`로 다르게 지정하면 하나의 창고에서 수십 개의 독립된 사이트를 찍어낼 수 있습니다.

> [!IMPORTANT]
> **주소(도메인)에 관하여**
> 기본적으로 부여되는 `어쩌구.vercel.app`은 공짜이므로 언제든 예쁘게 수정해서 쓸 수 있습니다. 단, `.vercel.app` 꼬리표를 떼려면 나중에 가비아 같은 곳에서 `대표님이름.com`을 만 원 정도에 구매해서 연결하시면 됩니다.

> [!WARNING]
> **게시판 / 결제 시스템이 필요해지면?**
> 지금의 예쁜 카탈로그(정적 사이트)를 넘어, 고객이 신용카드를 긁거나 스마트폰 사진을 직접 사이트에 업로드하게 만드시려면 **'백엔드(Backend)'와 '데이터베이스(DB)'**라는 거대한 보이지 않는 창고 시스템을 사이트 뒤에 붙여 연결하는 추가 공사가 필요합니다. 

이 노트 하나면 지금까지 하신 작업의 흐름이 한눈에 보이실 겁니다. 앞으로도 이것만 기억하시면 모든 IT 프로젝트를 대표님 지휘하에 마음대로 통제하실 수 있습니다! 👑
